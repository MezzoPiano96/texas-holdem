# 局域网联机德州扑克 · 操作手册

给完全没碰过命令行的人看的版本。整个过程不需要联网（不需要路由器/公网设置），只需要主机和所有玩家在**同一个WiFi**下。

## 你需要准备的东西

- 一台电脑当主机（Mac/Windows都行，下面以Mac为例）。
- 这台电脑上装好 [Node.js](https://nodejs.org/)（装一次就行，以后不用再装）。检查是否已经装过：

  打开"终端"（Terminal），输入：
  ```bash
  node -v
  ```
  如果显示一个版本号（比如 `v18.20.8`），说明已经装好了，跳过下一步。如果提示"command not found"，去 [nodejs.org](https://nodejs.org/) 下载安装包，装好之后重新打开终端再试一次。

## 第一次使用：装一次依赖

打开终端，输入（注意路径要换成你自己电脑上这个文件夹实际的位置）：

```bash
cd "/Users/nagi/Project/Texas Holdem/multiplayer"
npm install
```

这一步只要做一次。装完之后，这个文件夹里会多出一个 `node_modules` 文件夹，以后不用再重复这一步（除非你换了电脑）。

## 每次开局：怎么把服务器跑起来

1. **确认所有人都连同一个WiFi**（不是流量，不是各自的热点）。如果现场没有WiFi，主机可以开"个人热点"，让大家都连主机的热点。
2. **关掉VPN / 全局代理**（如果开着的话）。"智能代理/规则模式"一般没问题，但"全局代理"会让局域网连接失败——这点之前讨论过，记得检查一下。
3. 打开终端，输入：
   ```bash
   cd "/Users/nagi/Project/Texas Holdem/multiplayer"
   npm start
   ```
4. 看到类似下面的输出就说明启动成功了：
   ```
   德州扑克联机服务器已启动！
   让大家在同一个WiFi下，用手机浏览器打开下面的地址：
     http://172.18.3.154:3000/
   按 Ctrl+C 关闭服务器。
   ```
   把这条 `http://...:3000/` 的地址念给大家，或者发到群里。
5. **第一次启动**，Mac可能会弹出"是否允许 node 接受网络连接"——点"允许"，否则别人连不进来。
6. 大家在手机浏览器（Safari/Chrome都行）里输入这个地址，会看到"联机大厅"页面，输入名字点"加入"。
7. **如果主机自己也想玩**：在这台电脑上再开一个浏览器标签，访问同一个地址（或者直接用 `http://localhost:3000/`），跟其他人操作一样，输入名字加入座位。
8. 第一个加入大厅的人自动成为"房主"，会多出"添加AI / 调整人数 / 开始游戏"这些按钮。人数不够时可以用"添加AI"补位，凑够至少2人即可开始；房主想什么时候开始都可以，不用等所有座位坐满。
9. 玩完了，回到终端按 `Ctrl + C` 就能关掉服务器。

## 常见问题

**手机打不开那个地址 / 一直转圈**
- 先确认手机和电脑真的连的是同一个WiFi（注意有些路由器会把2.4G和5G分成两个不同名字的网络，要连同一个）。
- 确认手机没开VPN/全局代理。
- 终端里再跑一次 `npm start`，把打印出的地址重新核对一遍，IP地址错一个数字就连不上。

**牌局打到一半，有人的手机黑屏/锁屏了**
- 不用管，系统会等一小段时间（大概75秒）没人操作就自动帮他弃牌，牌局会继续，不会卡住整桌。
- 那个人锁屏的设备重新打开浏览器、重新访问同一个地址，会自动认回他原来的座位和筹码（前提是用的还是同一个浏览器标签，没有清除过浏览数据）。

**想换一批人重新开一局，要不要重启服务器**
- 一局打完（有人筹码归零、只剩一个人）之后，这一桌就算结束了，没法在原来的进程里直接开新的一桌。回到终端，`Ctrl+C` 关掉，再 `npm start` 重新起一个全新的桌子即可。

**这个会暴露在公网上让陌生人连进来吗**
- 不会。这套东西只在你自己的WiFi内部能被访问到，外面网络上的人原则上连不进来（除非你自己额外做了端口转发或者部署到公网，这个手册里讲的方式不涉及那些）。但同一个WiFi下的所有设备理论上都能看到这个地址，所以建议只在自己信任的家庭WiFi下用，不要在公共WiFi（咖啡店、酒店）上开。
- 服务器会校验玩家行动是否合法，并且页面会转义玩家输入的名字，避免名字里带HTML标签影响界面。

## 给维护者：跑测试

如果你改了规则、边池、重连或下注逻辑，建议在 `multiplayer` 目录下跑一次：

```bash
npm test
```

测试会覆盖牌型评估、A2345顺子、边池分层、日志名字转义、稀疏座位映射、玩家行动校验和静态文件路径防穿越，并检查单机版/联机版两份语音资源是否同步。

如果只想检查语音资源：

```bash
npm run check:voice
```
